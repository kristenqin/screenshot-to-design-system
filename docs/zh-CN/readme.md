# Screenshot to Design System

这个项目用于沉淀一个自动化工作流：从软件截图反推出可编辑设计稿和可复用设计系统。

核心判断是：

> 截图还原设计稿不是一次性生图任务，而是一个逐步反推结构、提取组件和 token、再重建页面的过程。

## 目标

- 把完整软件截图转成可编辑设计草稿。
- 尽量复用成熟工具，不从零重做每个环节。
- 自动发现重复 UI 模式，形成 component candidates。
- 生成设计系统，而不是只生成一张静态视觉图。
- 用 screenshot diff 和 OCR diff 作为迭代反馈。

## 当前判断

线稿生成阶段先把豆包生图 API 当作黑盒能力使用。当前它的效果足够好，MVP 阶段不值得重做。

真正的产品风险在中间层：

- UI 元素层级
- 可复用组件候选
- 颜色和字体 token
- 布局规则
- 可编辑输出
